# 灵犀 · 消息合并助手 (astrbot_plugin_chat_merger)

> 作者：灵犀 | 仓库：[astrbot_plugin_chat_merger](https://github.com/gongzhudeng/astrbot_plugin_chat_merger) | 主页：[gongzhudeng](https://github.com/gongzhudeng)

合并连续消息后统一发送给AI，模拟真人聊天节奏。支持AI思考时显示"正在输入"状态、图片触发超长等待、等待时间随机波动等。

## 功能说明

当你连续发送多条消息时，插件会自动等待一段时间，将消息合并后再统一发送给AI回复，避免AI每条消息都单独回复。

### 核心逻辑

1. **字数越少，等待越长** - 短消息给你更多时间继续输入
2. **字数越多，等待越短** - 长消息说明你已经说完了
3. **每次新消息重新计算** - 发新消息后倒计时重新开始
4. **AI思考时自动合并** - AI回复期间发的消息会攒着，等AI回复完再统一处理
5. **AI思考时显示"正在输入"** - AI处理消息时，QQ等平台会显示"对方正在输入..."状态
6. **图片触发超长等待** - 单独发图片（不带文字）自动进入超长等待模式
7. **等待时间随机波动** - 超长等待时间可设置随机范围，更自然

### 可用命令

| 命令 | 说明 |
|------|------|
| `/合并帮助` | 显示所有可用命令 |
| `/合并状态` | 查看当前队列状态（消息数、字数、AI是否在处理） |
| `/立即发送` | 立即发送队列中的消息，不等延迟 |
| `/清空队列` | 清空消息队列 |
| `/合并配置` | 查看当前所有配置项 |
| `/合并调试` | 切换调试模式（开启后系统日志会显示详细信息） |

### 配置说明

#### 延迟配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 最短等待时间 | 2秒 | 长消息的等待时间 |
| 最长等待时间 | 10秒 | 短消息的等待时间 |
| 短消息字数阈值 | 10字 | 低于此字数使用最长等待 |
| 长消息字数阈值 | 50字 | 单条超过此字数使用长消息延迟 |
| 长消息等待时间 | 2秒 | 单条超长消息的专用等待时间 |
| 消息条数阈值 | 10条 | 达到此条数直接发送 |

#### 跳过关键词

命中后直接发送，跳过所有延迟。支持多个关键词，可选择"包含"或"完全匹配"模式。

默认跳过词：急、紧急、马上、立刻

#### 超长等待关键词

触发后进入超长等待模式（默认5分钟），给你充足时间组织语言。支持多个关键词。

默认等待词：等一下、稍等、等我一下

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 等待时间 | 300秒 | 触发后的等待时间，设为0表示无限等待 |
| 等待时间随机变化范围 | 0秒 | 实际等待时间会在设定值±此范围内随机波动 |
| 图片触发超长等待 | 开启 | 单独发图片（不带文字）自动触发超长等待 |

#### AI忙感知

当AI正在处理上一条消息时，合并消息会等待AI处理完成后再发送，避免消息堆积。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用AI忙感知 | 开启 | AI忙碌时自动等待 |
| 检查间隔 | 3秒 | 每隔多久检查一次AI是否空闲 |
| 最大等待时间 | 120秒 | 超时后强制发送 |

#### 输入状态显示

AI思考时在QQ等平台显示"对方正在输入..."状态（仅私聊，需NapCat/aiocqhttp平台）。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用输入状态 | 开启 | AI思考时显示正在输入 |
| 刷新间隔 | 0.5秒 | 输入状态刷新频率 |

### 使用示例

```
你：你好
你：我想问个问题
你：关于Python的
→ 三条消息合并为一条发送给AI，AI统一回复

你：急，服务器崩了
→ 命中"急"，立即发送

你：等一下
→ 触发超长等待（默认5分钟，可设随机波动），给你时间组织语言

你：[图片]
→ 单独发图片触发超长等待，等待结束后和后续消息一起发送给AI

AI思考中...
→ QQ显示"对方正在输入..."状态
```
